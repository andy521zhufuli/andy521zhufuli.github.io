3. 设置虚拟主机映射

步骤如下

3.1 开启虚拟主机功能

使用任何编辑器打开/Applications/XAMPP/xamppfiles/etc/httpd.conf这个文件，原文中说看到的是这样的：

# Virtual hosts
#Include /Applications/XAMPP/etc/extra/httpd-vhosts.conf

但是我安装后的看到的这样的

# Virtual hosts
#Include etc/extra/httpd-vhosts.conf

其实我感觉这两个没什么区别，前者是绝对路径，后者是相对路径。但是这里配置还是按照文中说的配置，去掉前面的#，如下：

# Virtual hosts
Include /Applications/XAMPP/etc/extra/httpd-vhosts.conf

3.2 创建自己的虚拟主机

打开/Applications/XAMPP/xamppfiles/etc/extra/httpd-vhosts.conf这个文件，将<VirtualHost >修改为如下：

<VirtualHost *>
  # Change this to the domain which points to your host.
  ServerName 10.242.92.118

  # Change this to the path where you put 'phabricator' when you checked it
  # out from GitHub when following the Installation Guide.
  #
  # Make sure you include "/webroot" at the end!

  DocumentRoot /Users/andy/NetEaseWork/YanXuanProject/yx_code_review/phabricator/webroot

      <Directory "/Users/andy/NetEaseWork/YanXuanProject/yx_code_review/phabricator/webroot">
          Options Indexes FollowSymLinks Includes execCGI
          AllowOverride All
          Require all granted
      </Directory>

      RewriteEngine on
      RewriteRule ^/rsrc/(.*)     -                       [L,QSA]
      RewriteRule ^/favicon.ico   -                       [L,QSA]
      RewriteRule ^(.*)$          /index.php?__path__=$1  [B,L,QSA]
</VirtualHost>

注意事项：此文件中只保留一个VirtualHost标签，其他VirtualHost标签注释掉吧，不然可能不成功。

注释： 其中ServerName为自己ip地址（ifconfig查看）
DocumentRoot对应phabricator中webroot绝对路径。

保存，重启Apache。浏览器中输入自己IP地址访问。

3.3 访问验证

如果你比我幸运的话，你会在浏览器中看到Phabricator的配置mysql的页面

如果你像我一样唔好彩既话，会出现403，这里就要再次打开/Applications/XAMPP/xamppfiles/etc/httpd.conf文件，找到如下内容，将User改为你的用户名

# User/Group: The name (or #number) of the user/group to run httpd as.
# It is usually good practice to create a dedicated user and group for
# running httpd, as with most system services.
#
User andy 
Group daemon

ok，重启Apache，再次在浏览器输入自己的ip。

3.4 积攒人品的时候

如果你够幸运，恭喜你，到这里你就可以进入配置数据库的一步，如果你像我一样不能总是靠运气，而是只能靠颜值的话，你这里还需要一步



访问验证中的文件再次打开,找到第一个<Directory />标签

#
# Deny access to the entirety of your server's filesystem. You must
# explicitly permit access to web content directories in other
# <Directory> blocks below.
#
<Directory />
    AllowOverride none
    Require all denied
</Directory>

修改如下：

<Directory />
    AllowOverride none
    Order Deny,Allow
    Allow from all
    Require all granted
</Directory>

到这里，相信你跟我一样，也变成了上天眷顾的孩子，重启Apache，重新打开浏览器，输入自己ip，你就可以看到phabricator提示你配置mysql了。但是如果你还是没有看到的话，对不起，我也帮补了你了，删除，卸载，重启电脑，从头开始把，孩子

4. 配置mysql数据库

如果你是第一次配置mysql（之前没有配置mysql的用户名，密码），那么进入phabricator的目录运行如下命令：

# 由于mysql安装在本机，这里的mysql.host可填localhost或127.0.0.1或本机实际IP
$ ./bin/config set mysql.host localhost

$ ./bin/config set mysql.user root

$ ./bin/config set mysql.pass ""

$ ./bin/storage upgrade

如果你之前配置过mysql，那么直接运行下面命令

$ ./bin/storage upgrade

再次访问吧，顺利的话，就会看到phabricator创建账户的页面了，按照提示创建页面登录后的,如下所示：

配置

打开页面，你会看到左上角有13个提示，按照这么多提示一个一个去设置吧

1. Base URI Not Configured

./bin/config set phabricator.base-uri 'http://10.242.92.118/'

2. No Authentication Providers Configured

目的就是设置一些email的域，可以允许那些email注册。，跟着他的指引，设置一下就可以了

遇到错误，在执行./bin/phd start命令的时候，遇到这个错误The PHP extension 'pcntl' is not installed,解决办法如下：

使用 $ brew install homebrew/php/php55-pcntl 命令
